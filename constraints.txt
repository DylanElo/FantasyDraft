# Reproducible dependency pins for `requirements.txt`.
#
# `requirements.txt` intentionally keeps loose `>=` ranges as the abstract
# runtime contract. This file pins the exact resolved versions -- including
# transitive dependencies -- so the Docker image (and any other install of
# `requirements.txt`) builds identically every time instead of picking up
# whatever the newest compatible release happens to be on a given day.
#
# `pip install -c constraints.txt -r requirements.txt` only constrains
# packages requirements.txt already pulls in; it never adds a new package by
# itself, so this file is safe to keep even for platform-conditional
# transitive deps (e.g. `colorama`, a Windows-only dependency of `click`).
#
# Regenerate deliberately, in a clean virtualenv, after validating the new
# versions:
#   python -m venv .constraints-venv
#   .constraints-venv/bin/pip install --upgrade pip
#   .constraints-venv/bin/pip install -r requirements.txt
#   .constraints-venv/bin/pip freeze --local > constraints.txt
